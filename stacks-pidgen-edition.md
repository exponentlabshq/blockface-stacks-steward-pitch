# üá≥üá¨ CAP Theorem in Pidgen üá≥üá¨

**"My people, make I yan you small about something wey dey shake database world well well ‚Äì na CAP theorem. E get three things wey any distributed system dey try balance: Consistency, Availability, and Partition tolerance. Make I break am down for you:**

**1. Consistency (C):**
This one mean say, anytime you check data, everybody go see the same thing. No confusion, no wahala. If person A update data, person B no go see old thing ‚Äì everybody dey see the latest matter.

**2. Availability (A):**
This one mean say, anytime you ask for data, system go answer you. E no matter if some server don fall, e go try give you answer. You no go dey wait sotay system dey recover.

**3. Partition tolerance (P):**
This one na say, even if network break or servers no fit talk to each other, system still go dey run. E fit handle gbege wey network dey give.

**But here be the wahala ‚Äì you no fit get all three perfect at the same time. You fit only pick two:**

* **C + A** ‚Äì data dey consistent, system dey available, but if network break, e fit crash.
* **C + P** ‚Äì data dey consistent, system fit survive network wahala, but sometimes e no go answer you quick.
* **A + P** ‚Äì system dey available, system fit survive network wahala, but sometimes data no dey fully consistent.

**So my people, CAP theorem dey teach us say: anytime we dey design big system, we gats sabi which two things we dey prioritize, because the third one go always suffer small. Na so e be!"**
